■ Phase 5 — Upload Status/Retry UI
  - Upload progress frame (hidden until first upload)
  - Indeterminate progress bar during upload
  - Green bar + byte count on success
  - Red error text + Retry button on failure
  - Retry re-fires the exact same upload from stored args

■ Auto-Discovery — Sirona device finding
  - TCP port scan on local subnet (port 12837) — finds Sirona in ~1 second
  - 50 parallel threads, 0.3s timeout per host
  - Falls back to hardcoded SIRONA_IP if scan finds nothing
  - SIRONA_IP env var overrides discovery entirely
  - UDP discovery also available (src/protocol/udp.py) but Sirona doesn't
    respond to UDP probes on this network

───────────────────────────────────────────────────────────────────────────────
  SIRONA CONNECTION
───────────────────────────────────────────────────────────────────────────────

Current machine: 192.168.139.10
Sirona ORTHOPHOS XG: 192.168.139.170:12837

Protocol: SiNet2 / P2K (proprietary binary, reverse-engineered)
  - UDP discovery on port 1999 (broadcast probe) — implemented but Sirona
    doesn't respond to it on this network
  - TCP sessions on port 12837 (heartbeat, expose, scanline data)

Auto-discovery approach:
  1. TCP port scan on local subnet, port 12837, 50 parallel threads
  2. First host that accepts connection = Sirona
  3. Fallback to SIRONA_IP env var or hardcoded 192.168.139.170

Key files for protocol:
  src/protocol/udp.py      — UDP discovery (async + sync)
  src/protocol/tcp.py      — TCP framing
  src/protocol/packets.py  — Packet encoding/decoding
  src/protocol/constants.py — Wire constants
  src/core/discovery.py    — High-level discovery service
  hb_decoder.py            — SironaLiveClient (heartbeat, kV, scanlines)
  docs/protocol.md         — Full protocol reference
